# Brief for Iris — Roster UI: Team Member Thumbnails

**From:** Adama
**Project:** Team Photos

---

## What I Need

Update the team roster UI to display a thumbnail avatar for each team member in their roster row.

## Where the Images Live

Thumbnails are stored per theme at:
```
Projects/Team Photos/themes/<theme>/thumbnails/<name>.webp
```

For now, the active theme is `dossier`. The thumbnail path for a member is:
```
Projects/Team Photos/themes/dossier/thumbnails/adama.webp
Projects/Team Photos/themes/dossier/thumbnails/prospero.webp
... etc
```

Names are lowercase, matching the team member's name (e.g. `muybridge.webp`, not `Muybridge.webp`).

**Fallback:** If a member's named thumbnail does not exist (e.g. a newly hired member whose image hasn't been generated yet), fall back to:
```
Projects/Team Photos/themes/dossier/thumbnails/placeholder.webp
```

Note: thumbnails will be generated by Muybridge (Image Processing Specialist) as a separate task. The UI should be ready to display them, with the fallback handling the gap until they exist.

## Display Spec

- **Size:** 256×256px source, but display at a smaller size appropriate for a roster row — something like 48×48px or 56×56px. Use your judgment on what looks right.
- **Shape:** Circular crop in the UI (standard avatar treatment) using `border-radius: 50%` or equivalent
- **Placement:** Left-most cell in each roster row, before the name
- **Alt text:** Member's name

## Behaviour

- If the thumbnail file exists → display it
- If not → display `placeholder.webp`
- The fallback should be handled gracefully — no broken image icons

## Notes

- The thumbnail files don't exist yet (Muybridge is building the pipeline). Build against the expected paths and test with placeholder.webp while waiting.
- Keep the implementation simple — this is a static file serve, not a dynamic asset system.
